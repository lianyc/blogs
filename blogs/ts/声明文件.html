    声明文件

    12/21/2023 typescript声明文件

    # 声明文件

    以.d.ts结尾的文件

    • 声明文件的作用：为js代码提供类型声明

    • 声明文件书写的位置：

      • 放置到tsconfig.json配置中包含（include）的目录中

      • 放置到node_modules/@types文件夹中

      • 手动配置：compilerOptions：

            "typeRoots": [
      "node_modules/@types",
      "src/@types"
    ]

      • 与js代码所在目录相同，并且文件名也相同的文件，用ts代码书写的工程发布之后的格式

    # 编写声明文件

    # 自动生成

    工程使用TS开发，编译之后是js文件。

    如果发布的文件，需要其他开发者使用，可以使用声明文件来描述发布结果中的类型

    "declaration": true, // 生成声明文件
"declarationMap": false, // 生成声明文件映射
"sourceMap": true // 生成源码映射

    # 手动编写

    • 对已有的js书写的库，并且更改为ts成本较高
    • 对一些第三方库，他们使用js书写，且没有提供声明文件

    全局声明

    声明一些全局的对象、属性、变量

    declare const lian:{
  log:(msg:string)=>void,
  error:(msg:string)=>void,
}
// lian.log('lian')

    namespace:表示命名空间，可以将其认为是一个对象，命名空间中的内容，必须通过命名空间.成员名访问

    declare namespace lian {
  function log(msg:string):void;
  function error(msg:string):void;
}
// lian.log('lian')

    type TimeOut = () => void;
declare function lianTimeOut(callback:TimeOut,time:number):number;
// lianTimeOut(() => {}, 1000)

    模块声明

    declare module 'lodash' {
  export function chunk<T>(array:T[], n: number): T[][];
}

    三斜线指令

    在一个声明文件中，包含另一个声明文件

    /// <reference path="./lodash/index.d.ts" />

    # 发布

    • 使用ts开发：打包之后随文件夹一起发布到npm

    • 为其他第三方库开发的声明文件：发布到@types/**

      • Fork DefinitelyTyped (opens new window)
      • clone 到本地
      • types添加文件夹，开发，提交，推送到远程
      • 发送pull request
      • 等待官方审核，通过后合并并发布
