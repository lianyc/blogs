    11/15/2023 构建工具rollup

    # 1.rollup介绍 中文文档 (opens new window)

    • 开源类库优先选择

    • 以ESM标准为目标的构建工具

    • Tree Shaking: 去除不需要的代码，提升打包性能

      npm i -g rollup

      基础打包命令示例

      rollup -i index.js // 输出到命令行
rollup -i index.js --file dist.js // 输出到文件
rollup -i index.js --file dist.js --format umd // umd模式
rollup -i index.js --file dist.js --format cjs // cjs模式
rollup -i index.js --file dist.js --format es // es模式
rollup -i index.js --file dist.js --format iife // iife自执行函数模式
rollup -i index.js --file dist.js --enviroment MODE:development // 环境变量
rollup --help //帮助文档

      rollup命令

      rollup -v // 查看版本 --version
rollup -i // input 入口文件 --input
rollup -o // 输出文件 --file
rollup -f // 输出文件类型 --format (amd, cjs, es, iife, umd, system)
rollup -d // 输出文件目录 --dir
rollup -n //  --name 指定UMD挂载到全局的name
rollup -w //  --watch 实时监听文件变化并重新打包
rollup -c //  --config 指定配置文件
rollup --enviroment // 指定环境变量
rollup --plugin json // 指定插件

    # 2.rollup配置文件

    import resolve from "@rollup/plugin-node-resolve" // 将import的库一起打包进文件
import commonjs from "@rollup/plugin-commonjs" // 打包commonjs标准的文件
import json from '@rollup/plugin-json' // 处理json文件
import { terser } from 'rollup-plugin-terser' // 代码压缩
const mode = process.env.MODE
const isLocal = mode === "local"
export fefault [
{
    input: "index.js"，
    external: ['react'], // 排除一起打包的库，配合resolve使用
    //external: {
    //    'react': "React" // UMD名字
    //},
    plugins: [resolve(), commonjs(), json()], // 按照顺序执行
   	output: { // 也可以改为数组，输出多个文件
        file: "dist/index.umd.js",
        foramt: "umd",
        name: "Index",
        plugins: [terser()],  // 导出时用的插件
        banner: "/** Hello, this is banner **/" //打包完的文件最上面增加的注释
    }，
},{
    input: "index.js"，
    plugins: [json()],
   	output: {
        file: "dist/index.es.js",
        foramt: "es",
    }，
}]

    # 3. rollup插件

    通用配置

    • include
    • exclude

    官方插件地址 (opens new window)

    • alias (opens new window) 路径别名

      plugins: [
    alias({
      entries: [
        { find: 'utils', replacement: '../../../utils' },
        { find: 'batman-1.0.0', replacement: './joker-1.5.0' }
      ]
    })
    alias({
  		entries: {
    	utils: '../../../utils',
    	'batman-1.0.0': './joker-1.5.0'
  	  }
	});
]

    • babel (opens new window) 编译到不同版本js的编译工具

      plugins: [babel({ babelHelpers: 'bundled' })]

    • replace (opens new window) 将固定的变量进行替换

      plugins: [
    replace({
      'process.env.NODE_ENV': JSON.stringify('production'),
      __buildDate__: () => JSON.stringify(new Date()),
      __buildVersion: 15
    })
]

    常用插件

    • commonjs (opens new window) commonjs转换为ES6
    • babel (opens new window) 编译到不同版本js的编译工具
    • typescript (opens new window) typescript支持，只编译不报错
    • replace (opens new window) 将固定的变量进行替换
    • node resolve (opens new window) 可以使用node_modules里的第三方库
    • eslint (opens new window) 代码检查
    • image (opens new window) 使用图片
    • strip (opens new window) 去除log,debuger
    • wasm (opens new window) wasm支持
    • terser (opens new window) 代码压缩
    • rollup-plugin-typescript2 (opens new window) 第三方库，可以输出类型检查错误,比官方功能多一点
