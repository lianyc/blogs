    vite进阶

    11/14/2023 构建工具vite

    # 1.HMR热更新

    vite创建的vue项目默认开启热更新

    手动添加热更新逻辑

    export function render() {
    document.querySelecter("#app").innerHtml="<h1>hello vite</h1>"
}
if(import.meta.hot) {
    import.meta.hot.accept((newModule) => {
        newModule.render()
    })
}

    # 2.glob import:引入文件夹

    // 异步引入
const globModules = import.meta.glob('/glob/*') // 文件夹路径+* *.js  *.json *-[0-9].js
Object.entries(globModules).forEach(([k,v]) => {
    console.log(k,v)
    v.then((m) => {
        console.log(m.default)
    })
})
// 编译直接引入
const globModules = import.meta.globEager('/glob/*')

    依赖fast-glob (opens new window)库

    # 3.预编译

    • vite速度比较快的原因之一
    • node_modules/.vite文件夹下生成缓存
    • CommonJs to ESM
    • bundle files together

    排除或添加预编译文件：

    export default defineConfig({
	optimizeDeps: {
		include: ["react"], //手动添加
		exclude: ["vue", "loadsh-es"] // 手动排除
	}
})
