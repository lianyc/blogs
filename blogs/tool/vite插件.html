    # 1.vite插件 (opens new window)

    vite插件其实是受限制的rollup插件，支持rollup其中一部分功能

    命名

    • rollup-plugin-xxx 支持rollup和vite
    • vite-plugin-xxx 只支持vite

    vite兼容rollup的钩子

    • 服务启动时（调用一次）

      • options
      • buildStart

    • 每个模块

      • resolveId
      • load
      • transform

    • 服务关闭时

      • buildEnd
      • closeBundle

    modulePased不会被调用

    vite中配置rollup插件

    	build: {
      rollupOptions: {
        plugins: []
      }
    }

    vite插件兼容rollup的条件

    • 没有使用modulePased钩子
    • 打包钩子和输出钩子质检没有很强的耦合

    vite独有的钩子

    • config
    • configResolved
    • configureServer
    • transformIndexHtml
    • handleHotUpdate

    # 2.vite插件执行时机

    • pre
    • normal
    • post

    test.plugin.ts

    export default (enforce ?: 'pre' | 'post') => {
    return {
        name: 'test',
        enforce,
        buildStart() {
            console.log('build start, enforce:', enforce);
        },
        resolveId() {
            console.log('resolveId, enforce:', enforce);
        },
        load() {
            console.log('load, enforce:', enforce);
        }
    } 
}

    # 4.HMR API

    export interface ViteHotContext {
  readonly data: any // 只读，用于保存热更新之前的状态，重用到更新之后的模块

  accept(): void
  accept(cb: (mod: ModuleNamespace | undefined) => void): void
  accept(dep: string, cb: (mod: ModuleNamespace | undefined) => void): void
  accept(
    deps: readonly string[],
    cb: (mods: Array<ModuleNamespace | undefined>) => void,
  ): void

  acceptExports(
    exportNames: string | readonly string[],
    cb?: (mod: ModuleNamespace | undefined) => void,
  ): void

  dispose(cb: (data: any) => void): void // 清除副作用
  prune(cb: (data: any) => void): void
  invalidate(message?: string): void // 强制重新加载整个应用

  on<T extends string>(
    event: T,
    cb: (payload: InferCustomEventPayload<T>) => void,
  ): void
  send<T extends string>(event: T, data?: InferCustomEventPayload<T>): void
}
